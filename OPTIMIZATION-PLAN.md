# Loomscope — Optimization Plan (v2.0.0-rc.3)

Generated by a `/goal` run: static review + full test suite executed in an
**isolated Docker environment** on the r7 server (nothing installed into the
system env). Loomscope = visual viewer for Claude Code `.jsonl` transcripts
(React 18 + Vite 8 + TS, Hono/Node server, zustand, @xyflow/react canvas, SSE).

## P0 — Correctness & broken tooling (fix first; these block CI/build)

1. **[HIGH] `npm run build` is broken on a clean checkout.** `tsc --noEmit`
   fails on test-only type drift: `workflowHooks.test.tsx` builds a `SessionState`
   missing `lastTurnUserSubmittedAt/appliedVersion/serverVersion/rawAppliedRecordUuids`;
   `WorkflowSummary` fixtures use removed/renamed fields (`fileTouchCount`,
   optional `inputTokens`). *Fix:* update the fixtures to current types (a shared
   `makeSessionState()`/`makeWorkflowSummary()` test factory prevents recurrence),
   **or** split typecheck so `build` runs `tsc -p tsconfig.build.json` excluding
   `**/*.test.*`. Recommend both: keep tests typechecked in `npm run typecheck`,
   keep them out of the `build` gate.

2. **[HIGH] `npm run lint` cannot run.** No `eslint.config.*`/`.eslintrc*` and no
   eslint in `devDependencies`/`node_modules`. *Fix:* add `eslint` + a flat
   `eslint.config.js` (typescript-eslint + react-hooks), or delete the `lint`
   script and the orphaned `// eslint-disable` comments. The dead disable
   comments are actively misleading.

3. **[HIGH] e2e suite can't run anywhere but the author's machine.** Add
   `@playwright/test` to `devDependencies` (it's imported but absent — `vite.config.ts`
   even excludes `e2e/**` from vitest to dodge the missing import). Replace the
   hard-coded session/CWD and the `~/Agentloom` symlink with: (a) a committed
   small fixture transcript served via `--root`, and (b) data-driven discovery
   like the new `e2e-smoke/smoke.spec.ts`. Wire `webServer` into the Playwright
   config so it boots the server itself.

4. **[MEDIUM] Incremental-parse correctness** (`src/parse/jsonl.ts`):
   - `:325,351` re-uses cached records whenever `byteSize <= curSize`; a same-size
     in-place rewrite is missed (`mtimeMs` is captured but never compared). *Fix:*
     full reparse when `byteSize === curSize && mtimeMs` changed.
   - `:173,353` tail-reads with `createReadStream({encoding:"utf8", start:byteSize})`;
     if the previous boundary fell mid-codepoint the first tail line decodes
     corrupt. *Fix:* resume at last consumed-newline offset, or read Buffers with a
     stateful `StringDecoder`. (Both copies — `readRecordsIncremental` too.)

5. **[MEDIUM] Duplicate-delta forces full refresh** (`src/store/sessionSlice.ts:636-644`):
   any `seq !== appliedVersion+1` is treated as a gap, so an SSE reconnect replay
   (`seq <= appliedVersion`) triggers a ~4 s `refreshSession`. *Fix:* short-circuit
   `delta.seq <= appliedVersion` as a no-op. (`src/sse/signalClassifier.ts` already
   implements this suppression but is **never called in production** — wiring it in
   is the root fix.)

## P1 — Performance (large sessions: 600–1500 nodes, up to 256 MB jsonl)

6. **[HIGH] React Flow renders every node.** `ChatFlowCanvas.tsx:934` &
   `WorkFlowCanvas.tsx:326` omit `onlyRenderVisibleElements`; RF v12 only culls
   when opted in (the comment claiming auto-cull is wrong). *Fix:* add
   `onlyRenderVisibleElements={true}` (verify off-screen edge/ribbon geometry).

7. **[HIGH] Node cards aren't memoized + identity churn.** Cards
   (`ChatNodeCard.tsx`) lack `React.memo`, and `layoutDag.ts:286 refreshChatNodeContent`
   mints a fresh `data` object for *every* node on any change, so one streaming
   delta re-reconciles all N subtrees. *Fix:* `React.memo` the cards and preserve
   `data` identity for unchanged nodes.

8. **[HIGH] `ModelRibbonLayer.tsx:58-83`** is always mounted, subscribes to
   `transform` (fires every pan/zoom frame), and builds a Map over ~1500 nodes
   *before* the `if (!hoveredEdge) return null`. *Fix:* move the guard above the
   loop; only subscribe to `transform` while an edge is hovered.

9. **[HIGH] `removeSession` leaks per-session maps** (`sessionSlice.ts:1217`):
   clears only `sessions` + localStorage, leaving `tasksBySession`,
   `inflightBySession`, `rateLimitBySession`, `deferralBySession`, the 5
   `gitFilesSlice` maps, and `workflowViewports`. Every visited-then-removed
   session leaks for the app's life. *Fix:* clear all parallel maps by id.

10. **[HIGH] Incremental append copies the whole record array**
    (`jsonl.ts:168,347` `prevState.records.slice()`): O(N) per SSE event defeats
    incrementality on a 1.5 M-record session. *Fix:* structural-share / append-log.

11. **[MEDIUM] O(N²) index rebuilds:** `workflow-summary.ts:359` rebuilds
    `byId`/`byResultUserUuid` per llm_call; `jsonl.ts:1265 findScheduleWakeupAncestor`
    full-scans per scheduled fire; `jsonl.ts:735` rebuilds a Map per reuse bucket.
    *Fix:* hoist these indexes to one build per `buildChatFlow`.

12. **[MEDIUM] Unbounded client memory:** `rawAppliedRecordUuids` Set
    (`sessionSlice.ts:373`) and `workflowViewports` (`:1378`) grow for the session
    lifetime on long live tails. *Fix:* FIFO-cap both (reuse the `SENT_TEXT_CAP`
    pattern from `sdkChannelSlice`).

13. **[MEDIUM] Render-body scans:** `ChatFlowCanvas.tsx:619` recomputes
    `latestNodeId` via a full backward scan + several `nodes.find(...)` per render.
    *Fix:* memoize on `[nodes]`, build an id→node map.

14. **[MEDIUM] Bundle splitting:** dynamic-`import()` `react-markdown`+`highlight.js`
    (the 498 KB `MarkdownView` chunk) and split the 651 KB `index` chunk; both are
    only needed when a detail/markdown view opens.

## P2 — Security (Mode-A localhost model is careful; harden before any Mode-B)

15. **[MEDIUM] `/api/sessions/.../git/...` accepts an arbitrary `repo` path**
    (`routes/sessions.ts:602,669`) and runs `git -C <repo> show` — an
    arbitrary-local-repo read primitive if ever tunnel-exposed. *Fix:* cross-check
    `(repo,sha)` against tuples actually parsed from the session (the route comment
    already anticipates this). **Relevant to this user** — r7 runs a Cloudflare
    Tunnel; never expose Loomscope Mode-B without this.

16. **[MEDIUM] Broad CSRF bypass** (`middleware/csrf.ts:50-59`): `/api/sessions/`,
    `/api/fs/`, `/api/trash`, `/api/preferences` skip the token — including
    destructive ops (SDK turn dispatch, trash/purge, mkdir) — relying on CORS +
    localhost binding. *Fix:* pin the bypass list in a dedicated test; reconsider
    before 0.0.0.0 exposure. (Note `cors.ts:41` also lets no-`Origin` requests
    through — fine for Mode A, dangerous combined with the bypass under Mode B.)

## P3 — Testing & maintainability

17. **[HIGH] No tests on security/correctness-critical modules:**
    `middleware/csrf.ts` (the bypass allowlist), `routes/turns.ts`/`fork.ts`/
    `permissionPrompts.ts` (mutating/SDK/permission endpoints),
    `services/sessionWatcher.ts` (425-line live-update driver),
    `parse/raw-record.ts` & `parse/workflow-builder.ts` (foundational parsers).

18. **[MEDIUM] Fragile test patterns:** `app.test.ts` uses real `setTimeout`
    sleeps to bump mtime (flaky on coarse FS) and asserts traversal via HTTP 400
    only; `layoutDag.test.ts` hardcodes `NODE_HEIGHT` as `260` instead of importing
    the constant. Follow the `sse/*.test.ts` injected-clock style.

19. **[LOW] T3 — test noise:** something logs `ECONNREFUSED 127.0.0.1:3000` during
    `vitest run` (a test reaching a real socket). Stub/mocked it out; noisy logs
    hide real failures.

20. **[LOW] Dead code:** `liveEventSlice.ts:19,26,27` (unused `ssePending` +
    no-op subscribe stubs), `signalClassifier.ts` (tested but never called),
    `WorkFlowCanvas.tsx:195` (`const decoratedEdges = edges`), `jsonl.ts:1261`
    unused param; TODOs at `AttachmentCard.tsx:7`, `App.tsx:587`. Remove or wire up.

21. **[LOW] `clearDraftSession`** (`sessionSlice.ts:1208-1214`) uses a contrived
    conditional-type cast that defeats typechecking — replace with a plain
    `Partial<SessionSlice>` literal.

## Suggested order

- **Sprint 1 (unblock CI):** #1, #2, #3 — get build/lint/e2e runnable on any
  machine. Low risk, high leverage.
- **Sprint 2 (big-session UX):** #6, #7, #8, #9 — the four render/memory wins that
  matter most on 1000+ node sessions.
- **Sprint 3 (correctness):** #4, #5, #10, #11 — parse + delta robustness.
- **Ongoing:** P2 security before any Mode-B/tunnel exposure; P3 testing of the
  untested security-critical modules; clean up dead code.
